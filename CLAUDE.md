# Apple911 營運管理 App AI 開發規範

## 專案性質

本專案是 Apple911 手機維修店使用的營運管理 App。
本專案不是單純記帳 App，也不是購物車網站。

核心功能包含：

- 訂單管理
- 工單管理
- 商品管理
- 庫存管理
- 進貨紀錄
- 支出紀錄
- 報表系統
- 成本計算
- 利潤計算
- 庫存異動紀錄

技術架構：

- HTML
- Tailwind CSS
- JavaScript
- localStorage

## 庫存規則

- 工單使用零件時必須扣庫存
- 取消工單時必須補回庫存
- 庫存異動必須保留紀錄
- 盤點補正必須記錄原因
- 員工漏記、遺失報廢、盤點誤差必須可區分
- 通用型號邏輯不可破壞
- 保護貼、鏡頭貼可共用型號
- 充電線不可自動繼承手機型號
- 庫存低於安全數量時需要警示

## 成本與利潤規則

- 利潤 = 收款金額 - 零件成本 - 必要費用
- 零件成本必須獨立計算與顯示
- 現金不可扣 Line Pay 手續費
- 只有 Line Pay 才能扣手續費與營業稅
- 盤點誤差、損耗、補正成本必須可追蹤
- 商品移轉若有價差，必須記錄差額
- 報表需同步反映成本、損耗、盈虧

## 訂單 / 工單規則

- 新增訂單後畫面要回到最上方
- 訂單金額不可自動亂變
- 付款方式會影響利潤計算
- 工單使用的零件要能追蹤使用時間
- 訂單取消不能直接刪除庫存紀錄
- 訂單狀態修改要避免影響歷史報表

## 商品管理規則

- 商品管理要保持乾淨
- 新增商品應使用小按鈕、彈窗或收合區
- 不可讓大型新增表單直接佔滿主畫面
- 商品需支援品項、品牌、系列、型號
- 商品刪除按鈕必須可正常點擊
- 商品分類邏輯需清楚可維護

## 報表規則

- 報表要一眼看懂
- 圖表不可過大
- 手機版不可跑版
- 付款方式文字不可被切掉
- 營業額、零件成本、利潤要分開顯示
- 年度、月份、每日報表計算邏輯需一致

## UI 規則

禁止：

- 修改顏色
- 修改排版
- 修改字體
- 修改 icon
- 修改按鈕樣式

- 修改原有操作邏輯

任何未經要求的 UI 變動都視為錯誤。
